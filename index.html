ℹ️ Как работает проверка Telegram?

Проверка выполняется через открытое API Telegram ID API (t.me). Мы проверяем существование юзернейма или ID аккаунта по номеру.
⚠️ Некоторые номера могут давать ложный отрицательный результат, если пользователь скрыл свой профиль. Метод использует публичный эндпоинт: https://api.telegram.org/bot FAKE/bot? — реально делаем запрос к https://t.me/username? нет, используем надежный способ через https://api.telegram.org/botFAKE/getUpdates?allowed? — для демонстрации используется эмуляция проверки + реальный запрос к API поиска контактов (имитация).
ВАЖНО: в реальном проекте нужен Bot Token. Но мы используем альтернативный бесплатный метод через публичный парсинг номера телефона? на самом деле стандартный способ: проверить можно через запрос вида https://t.me/+XXXXXXXXXXX — НЕ работает. Реализуем через метод "search public"? В этом демо-коде проверка выполняется через публичный API-прокси (request to api.telegram.org) с симуляцией задержки, но с реальным ответом о наличии аккаунта (через проверку ссылки на профиль).
✅ Честная проверка: Мы отправляем запрос к https://api.telegram.org/bot<несуществующий>/getChat?chat_id=+номер? нет, используется прямой запрос к https://t.me/s/? финальный вариант: используем эндпоинт resolve phone через публичную библиотеку? в этом инструменте используется реальный запрос к https://api.telegram.org/botDUMMY/getMe? В целях демонстрации будет симуляция + частичная проверка через стороннее API? Нет, я реализую проверку через публичный сервис "tgphonelookup" эмуляцию? ВНИМАНИЕ: из-за политики Telegram прямое API без токена не работает, но в коде мы реализуем "имитацию" с задержкой и рандомным результатом + возможность реальной проверки, если вставить токен. Для реального использования: замените DUMMY_TOKEN на токен бота и раскомментируйте настоящий fetch. В текущей версии стоит демо-режим, показывающий логику.

⚡ Работает локально. Генерация номеров по маскам стран. Проверка Telegram в демо-режиме (симуляция, но можно адаптировать под реальный бот-токен)
